## 2026-05-07

**지시사항**: 세부기술 페이지 적용시기·사업화시설 표기 개선 (PR #6)

**구현 내용**:
- **적용일 → 적용시기** 라벨 변경, 경과기간 배지 표시 (예: `(4년 4개월)`)
  - `useAllData.js`에서 `apply_date` 기반 `elapsed_months` 실시간 계산 (CSV에는 미포함)
  - `App.css`에 `.elapsed-badge` 스타일 추가
- **매칭 시설 → 매칭 사업화시설** 라벨 변경
- **분야 공통 시설 매칭 로직 수정**: `item_no`가 없는 분야 공통 시설(예: 인공지능)이 같은 `sector_name`의 모든 기술에 매칭되도록 필터 조건 확장

**주요 결정사항**:
- `elapsed_months`는 CSV에 넣지 않고 프론트엔드에서 실시간 계산 — 매일 값이 바뀌므로 CSV에 저장하면 빌드 시점에 고정되어 부정확해짐

**오전 작업 — 분야 기술 목록 탐색 개선**:
- 분야 내 기술 목록에 도입 기간 필터 추가: `전체`, `3년 이상`, `5년 이상`
- 정렬 옵션 추가: `별표 순`(기본), `도입일 순`, `가나다 순`
- `폐지 포함` 체크박스로 폐지 기술 표시 기능 구현, 폐지 건수도 제목 옆에 표시
- 기술별 최초 도입시기를 프론트엔드에서 계산해 `YYYY-MM` 태그로 표시
  - 3년 이상은 연한 파란색, 5년 이상은 연한 노란색 태그로 구분
  - 필터 조건에 해당하지 않는 기술도 도입시기 태그는 항상 표시
- 분야 카드의 데이터셋 배지 아래에 분야 최초 도입 연도 표시 (`2022` 형식)
- 분야 카드에서 스크롤된 상태로 분야를 선택해도 기술 목록 상단으로 이동하도록 처리
- 필터 토글 및 목록 변경 시 가벼운 애니메이션 적용
- 인앱 브라우저로 UI 확인, `npm run lint`, `npm run build` 검증 완료

## 2026-05-08

**지시사항**: Claude 계획서 기반으로 세부기술 변경 연혁 diff 기능 구현

**구현 내용**:
- `historyKey`를 export해 TechDetail에서 동일 기술의 버전별 행을 재사용 그룹핑
- `diff` 패키지 추가, `diffWords` 기반 기술명/설명 단어 단위 변경 표시
- 세부기술 페이지의 매칭 사업화시설 아래에 접기/펼치기 가능한 변경 연혁 타임라인 추가
  - 신설 버전은 전체 내용 표시
  - 이후 버전은 상태 변경, 기술명 diff, 설명 diff 표시
  - 반복 재고시처럼 내용 변화가 없는 버전은 `내용 변경 없음` 표시
- 변경 연혁 상태 배지(신설/변경/삭제), 추가/삭제 diff 스타일, 모바일 간격 조정 추가

**검증**:
- `npm run lint`, `npm run build` 통과
- 인앱 브라우저에서 반도체 `차세대 메모리반도체` 3건 연혁 및 PIM/HBM diff 확인
- 신성장·원천 `주행상황 인지 센서 기술` 폐지 포함 상세에서 8건 연혁과 반복 삭제 `내용 변경 없음` 표시 확인

**오후 수정 — 사업화시설 매칭 오탐 수정**:
- 증상: 신성장·원천 `지능정보` 상세에서 `item_no`가 빈 시설 3건(`인지컴퓨팅`, `양자컴퓨터`, `스마트 물류`)이 모든 지능정보 기술에 기본 매칭됨
- 원인: 이전 매칭 조건이 `item_no` 없음 + 같은 `sector_name`이면 분야 전체 공통 시설로 간주해 너무 넓게 매칭
- 수정: 같은 번호/같은 기술명으로 매칭하거나, 시설명이 `공통`인 진짜 공통 시설만 분야 전체 매칭되도록 제한
- 확인: `언어이해 기술`은 사업화시설 0건, `양자컴퓨터 제작 및 활용 기술`은 자기 시설 1건, 전략기술 `인공지능` 공통 시설은 1건 유지
- `npm run lint`, `npm run build` 검증 완료

**추가 UX 수정 — 기술 목록 컨트롤 상태 유지**:
- 도입 기간 필터, `폐지 포함`, 정렬 상태를 `TechList` 내부가 아니라 `App`에서 보관하도록 변경
- 세부기술 상세에 들어갔다가 기술 목록으로 돌아오면 기존 선택 상태 유지
- 분야 목록으로 나가거나 새 분야를 선택하면 기본값(`전체`, 폐지 미포함, `별표 순`)으로 초기화
- 인앱 브라우저에서 유지/초기화 흐름 확인, `npm run lint`, `npm run build` 검증 완료
